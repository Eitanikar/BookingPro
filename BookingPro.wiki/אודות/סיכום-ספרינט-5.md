2. פוקוס הספרינט
----------------

הספרינט מתמקד בהשלמת **פיצ’רים קריטיים לחוויית משתמש מלאה** עבור לקוח וספק:
*   ניהול זמינות ולוחות זמנים
    
*   שליטה בתורים (צפייה / ביטול / היסטוריה)
    
*   הרחבת יכולות ניהול העסק (שירותים, שעות פעילות)
    
*   פיצ’רי אמון: דירוגים וביקורות
    
*   פיצ’ר אבטחה: איפוס סיסמה
    

* * *

3. מצב נוכחי
------------

*   כל ה־User Stories הוגדרו ונותחו
    
*   הספרינט מאורגן ומחולק ליחידות עבודה ברורות
    
*   קיימת תלות בין Backend ↔ Frontend במספר פיצ’רים (יומן, תורים)
    

* * *

4. אתגרים צפויים
----------------

*   לוגיקת זמן ותזמון ( זמינות, חופשת חנוכה)
    
*   שמירה על עקביות בין UI ליומן בצד השרת
    
*   טיפול נכון בהרשאות (לקוח מול ספק)
    

* * *

5. דגשים להמשך
--------------

*   להתחיל בפיצ’רים התשתיתיים (זמינות, שעות פעילות)
    
*   לבצע בדיקות ידניות לכל שינוי ביומן
    
*   לשמור על API ברור ואחיד לכל פעולות התורים
